# 多网卡网络配置指南

本文档记录了多网卡环境下同时访问 MES 内网和外部互联网的配置方法。

## 问题背景

### 环境描述
- 工控机有多个网卡
- 需要同时访问：
  - **MES 系统**：`172.24.10.21`（内网）
  - **外部互联网**：通过外部网卡上网

### 问题现象
插上外部上网网卡后，MES 连接失败，程序无法与 MES 通信。

### 问题原因
多网卡环境下存在多条默认路由，外部网卡的默认路由优先级更高（跃点数更小），导致所有未匹配的流量都走外网网卡，而外网网卡无法访问内部 MES 网络。

## 网络拓扑

```
┌─────────────────────────────────────────────────────────────┐
│                        工控机                                │
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │   以太网 11   │    │   以太网 5    │    │   其他网卡    │  │
│  │ 192.168.0.2  │    │172.25.118.90 │    │ 192.168.x.x  │  │
│  └──────┬───────┘    └──────┬───────┘    └──────────────┘  │
└─────────┼───────────────────┼───────────────────────────────┘
          │                   │
          ▼                   ▼
   ┌──────────────┐    ┌──────────────┐
   │  外网网关     │    │  内网网关     │
   │ 192.168.0.1  │    │172.25.118.1  │
   └──────┬───────┘    └──────┬───────┘
          │                   │
          ▼                   ▼
   ┌──────────────┐    ┌──────────────┐
   │   互联网      │    │  MES 服务器   │
   │  (8.8.8.8)   │    │172.24.10.21  │
   └──────────────┘    └──────────────┘
```

## 解决方案

### 核心思路
添加静态路由，指定 MES 网段（172.24.x.x）走内网网关，其他流量走外网网关。

### 执行命令

```powershell
# 添加永久静态路由（-p 参数表示永久，重启后仍有效）
route add 172.24.0.0 mask 255.255.0.0 172.25.118.1 -p
```

### 命令解释
| 参数 | 说明 |
|------|------|
| `172.24.0.0` | 目标网段（MES 所在网段） |
| `mask 255.255.0.0` | 子网掩码，匹配 172.24.x.x 所有地址 |
| `172.25.118.1` | 下一跳网关（内网网关） |
| `-p` | 永久路由，重启后保留 |

## 常用命令参考

### 查看网络配置

```powershell
# 查看所有网卡 IP 地址
Get-NetIPAddress -AddressFamily IPv4 | Format-Table InterfaceAlias, IPAddress, PrefixLength

# 查看启用的网卡
Get-NetAdapter | Where-Object Status -eq 'Up' | Select-Object Name, InterfaceDescription, Status, LinkSpeed

# 查看路由表
route print

# 查看默认路由
Get-NetRoute -DestinationPrefix '0.0.0.0/0' | Format-Table DestinationPrefix, NextHop, RouteMetric, InterfaceAlias
```

### 路由管理

```powershell
# 添加永久路由
route add <目标网段> mask <子网掩码> <网关IP> -p

# 删除路由
route delete <目标网段>

# 修改路由
route change <目标网段> mask <子网掩码> <新网关IP> -p

# 添加临时路由（重启后失效）
route add <目标网段> mask <子网掩码> <网关IP>
```

### 连接测试

```powershell
# 测试 MES 连接
ping 172.24.10.21

# 测试外网连接
ping 8.8.8.8

# 详细连接测试（包含端口）
Test-NetConnection -ComputerName 172.24.10.21 -Port 80

# 追踪路由路径
tracert 172.24.10.21
```

## 故障排查

### 问题1：MES 连接失败

**检查步骤：**
1. 确认路由是否存在
   ```powershell
   route print | findstr "172.24"
   ```
2. 确认内网网卡是否连接
   ```powershell
   ping 172.25.118.1
   ```
3. 重新添加路由
   ```powershell
   route add 172.24.0.0 mask 255.255.0.0 172.25.118.1 -p
   ```

### 问题2：外网无法访问

**检查步骤：**
1. 确认外网网卡状态
   ```powershell
   Get-NetAdapter | Where-Object Name -like "*以太网 11*"
   ```
2. 确认默认路由
   ```powershell
   Get-NetRoute -DestinationPrefix '0.0.0.0/0'
   ```
3. 检查外网网关连通性
   ```powershell
   ping 192.168.0.1
   ```

### 问题3：路由丢失（重启后）

如果使用了 `-p` 参数但路由仍丢失：
```powershell
# 检查永久路由
route print -4

# 重新添加永久路由
route add 172.24.0.0 mask 255.255.0.0 172.25.118.1 -p
```

## 注意事项

1. **网关 IP 变更**：如果内网网关 IP 变更，需要更新静态路由
2. **网卡名称变更**：Windows 更新或驱动更新可能导致网卡名称变化
3. **多 MES 服务器**：如果有多个 MES 服务器在不同网段，需要添加多条路由
4. **VPN 冲突**：如果使用 VPN，可能会覆盖路由表，需要注意

## 快速恢复脚本

如果配置丢失，运行以下命令快速恢复：

```powershell
# 恢复 MES 路由
route delete 172.24.0.0 2>$null
route add 172.24.0.0 mask 255.255.0.0 172.25.118.1 -p
Write-Host "MES 路由已恢复" -ForegroundColor Green

# 验证
ping 172.24.10.21 -n 1
ping 8.8.8.8 -n 1
```
