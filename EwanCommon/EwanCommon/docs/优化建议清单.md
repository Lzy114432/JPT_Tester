## 二、P0 - 必须修复

### 2.1 ⬜ LogicRunner.Dispose 不完整

**文件**: `EwanCore/StateMachine/LogicRunner.cs:234`

**问题**: 当前 Dispose 只设置标志位，不等待线程退出，可能导致资源泄漏或竞态条件。

**当前代码**:
```csharp
public void Dispose()
{
    _alive = false;
    RunTag = RunTimeTag.Stop;
}
```

**建议修改**:
```csharp
public void Dispose()
{
    if (!_alive) return;

    _alive = false;
    RunTag = RunTimeTag.Stop;

    // 等待线程退出，设置超时避免死锁
    if (_runThread != null && _runThread.IsAlive)
    {
        if (!_runThread.Join(TimeSpan.FromSeconds(2)))
        {
            // 超时警告（生产环境可记录日志）
            System.Diagnostics.Debug.WriteLine($"LogicRunner thread did not exit in time.");
        }
    }
}
```

### 2.3 ⬜ RequestAwaiter CancellationTokenSource 潜在泄漏

**文件**: `EwanCore/Messaging/RequestAwaiter.cs:39-73`

**问题**: 如果 `TrySetResult` 在超时前被调用，`CancellationTokenSource` 的 `token.Register` 回调可能未被正确清理。

**建议修改**:
```csharp
public Task<TResult> Register(TKey key, int timeoutMs = 30000, CancellationToken cancellationToken = default)
{
    var tcs = new TaskCompletionSource<TResult>(TaskCreationOptions.RunContinuationsAsynchronously);
    if (!_pending.TryAdd(key, tcs))
    {
        throw new InvalidOperationException($"Duplicate pending key: {key}");
    }

    if (timeoutMs < 0 && !cancellationToken.CanBeCanceled)
    {
        return tcs.Task;
    }

    var cts = timeoutMs >= 0
        ? CancellationTokenSource.CreateLinkedTokenSource(cancellationToken)
        : null;
    cts?.CancelAfter(timeoutMs);

    var token = cts?.Token ?? cancellationToken;
    CancellationTokenRegistration registration = default;

    if (token.CanBeCanceled)
    {
        registration = token.Register(() =>
        {
            if (_pending.TryRemove(key, out var removed))
            {
                removed.TrySetCanceled();
            }
        });
    }

    // 任务完成后清理资源
    tcs.Task.ContinueWith(_ =>
    {
        registration.Dispose();
        cts?.Dispose();
    }, TaskScheduler.Default);

    return tcs.Task;
}
```

## 三、P1 - 重要优化

### 3.1 ⬜ AlarmService 查找性能优化

**文件**: `EwanCore/AlarmSystem/AlarmService.cs:90`

**问题**: 使用 LINQ `FirstOrDefault` 在热路径上查找，报警数量多时性能下降。

**当前代码**:
```csharp
var existing = _alarms.FirstOrDefault(a => string.Equals(a.Key, key, StringComparison.Ordinal));
```

**建议修改**:
```csharp
public sealed class AlarmService : IAlarmService
{
    private readonly object _syncRoot = new object();
    private readonly List<Alarm> _alarms = new List<Alarm>();
    private readonly Dictionary<string, Alarm> _alarmsByKey = new Dictionary<string, Alarm>(StringComparer.Ordinal);

    public void AddAlarm(Alarm alarm)
    {
        if (alarm == null) throw new ArgumentNullException(nameof(alarm));

        AlarmChangeKind kind;
        Alarm changedAlarm;

        lock (_syncRoot)
        {
            var key = alarm.Key ?? string.Empty;
            if (_alarmsByKey.TryGetValue(key, out var existing))
            {
                existing.Touch(DateTime.Now, content: alarm.Content, level: alarm.Level,
                               unit: alarm.Unit, needReset: alarm.NeedReset, owner: alarm.Owner);
                kind = AlarmChangeKind.Updated;
                changedAlarm = existing;
            }
            else
            {
                _alarms.Add(alarm);
                _alarmsByKey[key] = alarm;
                kind = AlarmChangeKind.Added;
                changedAlarm = alarm;
            }
        }

        OnAlarmChanged(kind, changedAlarm);
    }

    public bool RemoveByKey(string key)
    {
        // 同步更新 _alarms 和 _alarmsByKey
        lock (_syncRoot)
        {
            if (_alarmsByKey.TryGetValue(key, out var alarm))
            {
                _alarms.Remove(alarm);
                _alarmsByKey.Remove(key);
                // ...
            }
        }
    }

    public void Clear()
    {
        lock (_syncRoot)
        {
            _alarms.Clear();
            _alarmsByKey.Clear();
        }
        // ...
    }
}
```

### 3.2 ⬜ LogicBase 字符串拼接优化

**文件**: `EwanCore/StateMachine/LogicBase.cs:79-81`

**问题**: 循环中使用 `+=` 拼接字符串，产生大量临时对象。

**当前代码**:
```csharp
var son = string.Empty;
for (var i = 0; i < _curSonLogicList.Count; i++)
{
    son += _curSonLogicList[i]?.ToString() ?? string.Empty;
}
```

**建议修改**:
```csharp
public virtual string GetLogicState()
{
    lock (_listLock)
    {
        if (_curSonLogicList.Count == 0)
        {
            return $"【{this}:{SwitchIndex}】";
        }

        var sb = new StringBuilder();
        foreach (var logic in _curSonLogicList)
        {
            if (logic != null)
            {
                sb.Append(logic.ToString());
            }
        }

        return sb.Length > 0
            ? $"【 {this}:{SwitchIndex}-{sb} 】"
            : $"【{this}:{SwitchIndex}】";
    }
}
```

**文件**: `EwanCore/Messaging/MessageBus.cs`, `EwanCore/Messaging/MessageBusExtensions.cs`

```csharp
// 订阅全部消息（建议只做轻量日志/指标）
using var all = MessageHub.Current.SubscribeAll(m => Console.WriteLine(m.GetType().Name));

// 带过滤条件的订阅（示例）
using var sub = MessageHub.Current.Subscribe<RecipeRead>(
    m => m.RecipeName.StartsWith("R"),
    m => Console.WriteLine($"recipe={m.RecipeName}, batch={m.Batch}"));
```

### 4.3 ⬜ RequestAwaiter 添加重试支持

**文件**: `EwanCore/Messaging/RequestAwaiter.cs`

**建议添加**:
```csharp
/// <summary>
/// 带重试的注册（适用于网络不稳定场景）。
/// </summary>
public async Task<TResult> RegisterWithRetry(
    TKey key,
    Func<Task> sendRequest,
    int retryCount = 3,
    int timeoutMs = 3000,
    CancellationToken cancellationToken = default)
{
    Exception lastException = null;

    for (var i = 0; i <= retryCount; i++)
    {
        var attemptKey = i == 0 ? key : /* 生成新 key */;
        try
        {
            var task = Register(attemptKey, timeoutMs, cancellationToken);
            await sendRequest();
            return await task;
        }
        catch (TaskCanceledException ex)
        {
            lastException = ex;
            // 继续重试
        }
    }

    throw new TimeoutException($"Request failed after {retryCount + 1} attempts", lastException);
}
```

### 5.3 ⬜ 使用 IAsyncEnumerable 改进消息消费

**文件**: `EwanCore/Messaging/MessageBus.cs`

**建议**（仅 .NET Core 3.0+ 场景，`EwanCommon` 当前为 net48）：

- 若迁移到 .NET（Core/5+），可考虑用 `Channel<T>` 替代 `BlockingCollection`，并提供 `IAsyncEnumerable<IMessage>` 作为可选消费方式。

**示例思路**（伪代码）:
```csharp
public async IAsyncEnumerable<IMessage> ConsumeAsync(
    [EnumeratorCancellation] CancellationToken cancellationToken = default)
{
    while (!cancellationToken.IsCancellationRequested)
    {
        IMessage msg;
        try
        {
            msg = await _channel.Reader.ReadAsync(cancellationToken);
        }
        catch (OperationCanceledException)
        {
            yield break;
        }

        yield return msg;
    }
}
```
